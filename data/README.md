# Development Data Directory

Keep everything you need for development purposes inside this folder, so that it's not inadvertedly pushed through git.
You can create the following folders:

```
data/
├── models/
└── configs/
```

## Models

Use this folder as binding target for the docker volume that will contain the models you want to use for inference.
This will be handled by the API.

## Configs

Use this folder to debug the dynamic configuration files that are generated by the API on service creation.
